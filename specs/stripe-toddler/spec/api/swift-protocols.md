# iPad App Swift Protocols

This document defines the core Swift interface contracts (Protocols) that structure the iOS POS application. They ensure clean separation of concerns between hardware communication (Bluetooth scanner and Stripe reader), network services, and local UI coordination.

---

## 1. Barcode Scanner Interface (Bluetooth HID Wedge)

Since the barcode scanner runs in Keyboard Wedge Mode (HID), it injects input text events globally. The application requires a keyboard event interceptor that aggregates characters and triggers a delegate call upon receiving a carriage return (`\r` or `\n`).

```swift
import Foundation

/// Represents a scanned inventory item identifier.
public struct ScannedBarcode {
    public let value: String
    public let timestamp: Date
}

/// Protocol to receive scanned barcode values from the event interceptor.
public protocol BarcodeScannerDelegate: AnyObject {
    /// Triggered when a complete barcode is successfully scanned and aggregated.
    /// - Parameter barcode: The ScannedBarcode value structure.
    func didScanBarcode(_ barcode: ScannedBarcode)
    
    /// Triggered when scanner keystrokes are received but are malformed or timed out.
    /// - Parameter error: Explanatory error details.
    func didEncounterScannerError(_ error: Error)
}

/// Service that coordinates hardware keystroke events.
public protocol BarcodeScannerServiceProtocol: AnyObject {
    var delegate: BarcodeScannerDelegate? { get set }
    
    /// Starts capturing global keystroke events (typically bound to the key window or an invisible input helper view).
    func startListening()
    
    /// Stops capturing keystroke events.
    func stopListening()
}
```

## 2. Stripe Terminal Manager Interface

The app integrates the physical Stripe Reader M2 via Stripe's Terminal iOS SDK. This manager encapsulates the SDK lifecycle.

```swift
import Foundation

/// States for the physical reader connection.
public enum ReaderConnectionState {
    case disconnected
    case scanning
    case connecting
    case connected(readerName: String, batteryLevel: Float)
}

/// Delegate that notifies the application state machine of Stripe Reader actions.
public protocol StripeTerminalManagerDelegate: AnyObject {
    func terminalManager(_ manager: StripeTerminalManagerProtocol, didChangeState state: ReaderConnectionState)
    func terminalManager(_ manager: StripeTerminalManagerProtocol, didEncounterError error: Error)
    func terminalManagerDidCompletePayment(_ manager: StripeTerminalManagerProtocol, paymentIntentId: String)
}

/// Protocol wrapping the Stripe Terminal iOS SDK operations.
public protocol StripeTerminalManagerProtocol: AnyObject {
    var delegate: StripeTerminalManagerDelegate? { get set }
    var connectionState: ReaderConnectionState { get }
    
    /// Scans for and automatically connects to the Reader M2 using BLE.
    func connectToReader()
    
    /// Disconnects from the active reader.
    func disconnectReader()
    
    /// Coordinates transaction payment. Runs through gathering card details and sending processing commands.
    /// - Parameters:
    ///   - amount: The transaction total in cents.
    ///   - clientSecret: The client secret of the payment intent generated by the backend.
    func collectPayment(amount: Int, clientSecret: String)
}
```

## 3. Backend API Client Interface

Coordinates REST calls to the Cloudflare Worker. It must handle challenge-response signing for **Apple App Attest** on every POS request.

```swift
import Foundation

/// Represents the localized models returned by the API client.
public struct POSInventoryItem: Codable, Identifiable {
    public var id: String { barcode }
    public let barcode: String
    public let name: String
    public let priceCents: Int
    public let imageUrl: URL
}

public struct PaymentIntentResponse: Codable {
    public let paymentIntentId: String
    public let clientSecret: String
}

public struct CaptureResponse: Codable {
    public let status: String
    public let transactionId: String
}

/// Interface defining POS REST operations.
public protocol BackendAPIClientProtocol: AnyObject {
    /// Configures/attests the device with the Cloudflare backend using Apple App Attest.
    /// Needs to be executed once during initial app boot or when credentials expire.
    func registerDeviceWithAppAttest() async throws
    
    /// Looks up a single item in the remote KV store by barcode.
    /// Implicitly signs the request payload using the registered App Attest Key.
    /// - Parameter barcode: The scanned barcode string.
    func fetchItem(barcode: String) async throws -> POSInventoryItem
    
    /// Requests Stripe Terminal Connection Token from backend.
    func fetchTerminalConnectionToken() async throws -> String
    
    /// Creates a PaymentIntent configured for the Terminal.
    /// - Parameters:
    ///   - amountCents: Payment amount.
    ///   - items: The list of items in the cart.
    func createPaymentIntent(amountCents: Int, barcodes: [String]) async throws -> PaymentIntentResponse
    
    /// Commands the backend to capture the authorized PaymentIntent and write transaction to D1.
    /// - Parameters:
    ///   - paymentIntentId: The target Stripe PaymentIntent ID.
    ///   - totalCents: Total amount.
    ///   - items: Details of the items sold for database logging.
    func captureTransaction(paymentIntentId: String, totalCents: Int, items: [POSInventoryItem]) async throws -> CaptureResponse
}
```

## 4. UI View Model & POS Coordinator Interface

Drives view layouts and orchestrates state changes in the main controller.

```swift
import Foundation

/// Defines POS user interface flow states.
public enum POSFlowState {
    case waitingForScan                   // Default screen. Bright colors, awaits scanner input.
    case cartActive(items: [POSInventoryItem], totalCents: Int) // Cart has items. Displays names, photos, prices.
    case readerSyncing                    // Connecting to Stripe Reader M2 or generating intents.
    case awaitingCardTap                  // Terminal prompt shown on UI. Toddler taps card.
    case processingPayment                // Capturing payment via Worker.
    case celebrating(itemsSold: [POSInventoryItem]) // Play band animation, fireworks, and sound effects.
    case error(message: String)           // Error feedback screen.
}

/// Interface for the view model binding to the SwiftUI / UIKit View layer.
@MainActor
public protocol POSViewModelProtocol: ObservableObject {
    var state: POSFlowState { get }
    var scannerConnected: Bool { get }
    var readerConnected: Bool { get }
    
    /// Triggered when the barcode scanner reads a code.
    func handleBarcodeScanned(_ barcode: String)
    
    /// Removes a specific index item from the active cart.
    func removeItem(at index: Int)
    
    /// Initiates checkout and Stripe Terminal collection flow.
    func startCheckout()
    
    /// Returns the UI to the empty cart scan screen.
    func resetPOS()
}
```
