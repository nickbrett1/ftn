Project Overview: Autonomous E-Commerce Agents
Objective
Build a system of autonomous LLM-driven agents that navigate fintechnick.com, simulate human shopping behaviors, and complete purchases using Stripe's test mode environment. The goal is to generate test payment volume, validate Stripe webhooks, and gain hands-on experience with agentic workflows and browser automation.

Architecture & Tech Stack
Language: Python

Orchestration: Dagster

Agent Framework: LangGraph or CrewAI

Browser Automation: Playwright (Python async API)

Payment Processing: Stripe API (Test Mode)

Target Environment: A dynamic frontend (handling standard modern DOM structures like those generated by SvelteKit or React).

Core Components

1. The Orchestrator (Dagster)
   Acts as the heartbeat and scheduler for the system.

Defines the agent runs as assets or jobs.

Schedules randomized or cron-based execution to simulate organic traffic.

Logs success, failure, and execution traces for each agent run.

2. The Agent Brain (LLM Framework)
   Controls the decision-making process.

Assigns specific personas to each run (e.g., "Bargain Hunter", "Erratic Shopper", "High-Roller").

Translates high-level goals ("Buy a cheap item") into step-by-step navigation instructions.

Parses the current state of the DOM to decide the next action.

3. The Hands (Playwright)
   Executes the agent's decisions in a headless browser.

Navigates to fintechnick.com.

Locates elements (buttons, forms, links) based on the LLM's interpretation of the page.

Handles asynchronous page loads, dynamic rendering, and clicks.

4. The Wallet (Stripe Integration)
   Handles the transaction simulation.

Inputs fake customer data (name, email, address) into the checkout form.

Injects Stripe's official test card credentials (4242 4242 4242 4242, any future date, any CVC) into the Stripe Elements iframe.

Submits the payment and awaits the success confirmation page.

Implementation Phases
Phase 1: Local Foundation & Playwright Setup
Initialize the Python project environment and install dependencies.

Create a basic Playwright script to launch a headless browser and navigate to the target URL.

Write helper functions to extract the page's visible text or HTML structure to feed to the LLM.

Phase 2: Agent Logic & DOM Interaction
Integrate the LLM framework (LangGraph/CrewAI).

Define the prompt templates and personas.

Build the feedback loop: Playwright grabs page state -> LLM decides action -> Playwright executes action (click, type, scroll).

Test the agent's ability to successfully navigate from the homepage to the final checkout screen without hardcoded selectors.

Phase 3: Stripe Checkout Automation
Implement the logic to handle the specific UI of the checkout page.

Program the Playwright script to specifically target and fill the Stripe Elements iframes (which requires specific frame-handling logic in Playwright).

Verify that successful test payments appear in the Stripe Developer Dashboard.

Phase 4: Dagster Orchestration
Wrap the Python scripts into Dagster software-defined assets or operations.

Set up a local Dagster instance.

Create a schedule to trigger the agent job at designated intervals.

Implement error handling to ensure a failed Playwright run gracefully exits and logs the failure in Dagster.
